# 개선일지: 스케줄 생성 알고리즘 부하 테스트 및 성능 분석

## 1. 개요
기존 프로젝트의 코어 로직인 `ScheduleGenerator`의 자원 효율성 및 성능을 검증하기 위해 Docker 기반의 부하 테스트를 진행함.

## 2. 테스트 환경
- **Runtime**: Docker (gradle:7.3.1-jdk17)
- **대상 로직**: `ScheduleGenerator.java` (재귀 탐색 기반 스케줄 추천 알고리즘)
- **테스트 도구**: JUnit 5 기반 커스텀 로드 테스트 (`ScheduleGeneratorLoadTest.java`)

## 3. 테스트 시나리오 및 결과

### 시나리오 A: 기초 부하 (Small Scale)
- **조건**: 1주일, 1개 교대조(9to6), 10명 신청, 1,000회 반복
- **결과**:
  - **P50**: 1.908 ms
  - **P95**: 32.669 ms
  - **P99**: 62.737 ms

### 시나리오 B: 현실적 알바 규모 (Realistic Alba Scale)
- **조건**: 1주일, 3개 교대조(Opening/Mid/Closing), 20명 신청(인당 10개 슬롯 랜덤 신청), 1,000회 반복
- **결과**:
  - **P50**: 10.340 ms
  - **P95**: 47.329 ms
  - **P99**: 75.760 ms

## 4. 분석 및 관찰
- **성능 특성**: 현실적인 알바 규모(20명 내외)에서도 중위값 10ms 내외의 매우 빠른 성능을 유지함. 교대조가 늘어남에 따라 탐색 속도가 시나리오 A 대비 약 5배 정도 증가했으나, 여전히 사용자에게는 즉각적인 응답이 가능한 수준임.
- **안정성**: P99 기준 75ms 정도로, 네트워크 지연 시간을 포함하더라도 0.1~0.2초 이내에 결과 확인이 가능함. 이는 백트래킹 엔진이 현실적인 제약 조건(신청 슬롯 제한 등) 하에서 효율적으로 동작하고 있음을 의미함.
- **규모 적정성**: 50명 이상의 대규모 조직보다는 10~30명 규모의 일반적인 편의점, 카페, 음식점 환경에 최적화된 성능을 보여줌.

## 5. 결론
개선된 현재의 알고리즘은 일반적인 아르바이트/소규모 사업장 환경에서 충분한 성능을 발휘함. 추후 대규모 조직 대응을 위한 확장성 테스트를 지속할 예정.
